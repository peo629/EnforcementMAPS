# EnforcementMAPS — Standalone Expo Application

> **CIVIC MAPS** · Centralised Integrated Visibility & Information for Compliance — Municipal Administration, Patrol & Supervision

## Overview

EnforcementMAPS is the officer field application for the CIVIC MAPS platform. This is a **standalone** Expo React Native application with all shared dependencies inlined — no monorepo workspace dependencies.

### Features

- **Patrol Zone Map** — Interactive map with Melbourne CBD patrol zones, street overlays, and officer markers
- **Code 21 Dispatch** — Receive and acknowledge dispatch requests with real-time push notifications
- **Zone Assignment** — View and manage patrol zone assignments
- **Officer Presence** — Online presence tracking and device registration
- **Location Pulse** — GPS location sharing with command/dispatch
- **Section Assignment Board** — View and self-assign to patrol sections
- **Authentication** — JWT-based auth with SecureStore token management

## Initial Setup

### 1. Clone and Install

```bash
git clone https://github.com/techcorp-DevApps/MAPS.git
cd MAPS
git checkout standalone/enforcement-maps

# Install dependencies (also applies patches automatically via pnpm)
pnpm install
```

### 2. Initialise EAS Project

This creates a new EAS project and generates a unique project ID.

```bash
# Log in to your Expo account
eas login

# Initialise the EAS project — this generates a new EAS_PROJECT_ID
eas init

# Note the project ID printed in the output. You need it for the next step.
```

After running `eas init`, update `app.config.ts` with your new project ID by setting the `EAS_PROJECT_ID` environment variable (see Environment Configuration below), or by replacing the `projectId` value directly in the `extra.eas` section.

### 3. Deploy PostGIS Database

The API server requires a PostgreSQL database with the PostGIS extension. Deploy one using your preferred provider:

**Railway (recommended):**
```bash
# Install Railway CLI
npm install -g @railway/cli

# Log in
railway login

# Create a new project
railway init

# Add a PostgreSQL database with PostGIS
railway add --plugin postgresql

# Enable PostGIS extension
railway connect
# Then run in psql: CREATE EXTENSION IF NOT EXISTS postgis;

# Get the connection string
railway variables
# Note the DATABASE_URL value
```

**Docker (local development):**
```bash
docker run -d \
  --name enforcement-maps-db \
  -e POSTGRES_USER=enforcement \
  -e POSTGRES_PASSWORD=your_secure_password \
  -e POSTGRES_DB=enforcement_maps \
  -p 5432:5432 \
  postgis/postgis:16-3.4

# Verify PostGIS is available
docker exec enforcement-maps-db psql -U enforcement -d enforcement_maps \
  -c "CREATE EXTENSION IF NOT EXISTS postgis; SELECT PostGIS_Version();"
```

### 4. Deploy API Server

Deploy the MAPS API server that this app connects to. Once deployed, note the server URL — you'll need it for environment configuration.

### 5. Environment Configuration

Copy the environment template and fill in your values:

```bash
cp .env.example .env
```

Then configure `eas.json` with your deployment URLs. Replace the empty `EXPO_PUBLIC_API_BASE_URL` and `EXPO_PUBLIC_DOMAIN` values in each build profile:

```bash
# Example: If your API server is at https://my-maps-api.up.railway.app
# Update eas.json development profile:
#   "EXPO_PUBLIC_API_BASE_URL": "https://my-maps-api.up.railway.app"
#   "EXPO_PUBLIC_DOMAIN": "my-maps-api.up.railway.app"
#
# Repeat for preview and production profiles with their respective URLs.
```

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `EXPO_PUBLIC_API_BASE_URL` | **Yes** | Full URL to your API server (e.g., `https://my-maps-api.up.railway.app`) |
| `EXPO_PUBLIC_DOMAIN` | Fallback | API server domain without protocol (used if `API_BASE_URL` is not set) |
| `EXPO_PUBLIC_ENVIRONMENT` | Yes | Environment name: `development`, `preview`, or `production` |
| `APP_ENV` | Yes | Build environment flag (matches `EXPO_PUBLIC_ENVIRONMENT`) |
| `EAS_PROJECT_ID` | **Yes** | Generated by `eas init` — required for EAS builds |
| `GOOGLE_MAPS_API_KEY` | **Yes** | Android Google Maps API key ([get one here](https://console.cloud.google.com/apis/credentials)) |
| `APP_FAMILY` | No | Platform identity (default: `CIVIC_MAPS`) |
| `APP_PRODUCT` | No | Product identity (default: `ENFORCEMENT_MAPS`) |

> **Note:** `EXPO_PUBLIC_API_BASE_URL` and `EXPO_PUBLIC_DOMAIN` are configured per build profile in `eas.json`, not in `.env`. The `.env` file is for local `expo start` development only.

```bash

eas build --platform android --profile development

# Build development client for iOS simulator
eas build --platform ios --profile development
```

### Preview Build (APK — internal testing)

```bash
# Build preview APK for Android
eas build --platform android --profile preview

# Build preview IPA for iOS (internal distribution)
eas build --platform ios --profile preview
```

### Production Build

```bash
# Build production AAB for Android (Google Play)
eas build --platform android --profile production

# Build production IPA for iOS (App Store)
eas build --platform ios --profile production
```

### Submit to Store

```bash
# Submit Android build to Google Play (requires google-play-service-account.json)
eas submit --platform android --profile production

# Submit iOS build to App Store
eas submit --platform ios --profile production
```

> **Google Play:** Place your service account key at `./google-play-service-account.json` before submitting. See [EAS Submit docs](https://docs.expo.dev/submit/android/) for setup instructions.

## Authentication

- Registration is restricted to `gov.au` and `melbourne.vic.gov.au` email domains
- Auth token stored as `patrol_auth_token` in SecureStore (native) or localStorage (web)
- JWT bearer token authentication with the MAPS API

## Roles

| Role | Description |
|------|-------------|
| `OFFICER` | Field officer (primary mobile app user) |
| `ADMIN` | Full access |
| `DISPATCHER` | Create/manage Code21, assign officers |
| `SUPERVISOR` | Read + limited write |

## Deployment Checklist

Use this checklist when deploying for the first time:

- [ ] Clone repo and checkout `standalone/enforcement-maps`
- [ ] Run `pnpm install`
- [ ] Run `eas login` and `eas init` to generate a new project ID
- [ ] Set `EAS_PROJECT_ID` in your environment or update `app.config.ts`
- [ ] Deploy PostGIS database and note the connection string
- [ ] Deploy API server connected to your PostGIS database
- [ ] Obtain a Google Maps API key and set `GOOGLE_MAPS_API_KEY`
- [ ] Update `eas.json` with your API server URLs in all build profiles
- [ ] Run `eas build --platform android --profile development` for first test build
- [ ] Verify the APK connects to your API server and authenticates successfully
- [ ] Run `eas build --platform android --profile production` for production build
